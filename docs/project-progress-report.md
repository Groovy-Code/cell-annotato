## 一、项目概述

**cell-annotato** 是一个单细胞转录组分类与专家注释修正平台（Web MVP），作为生物信息学毕业项目。

核心能力：用户上传单细胞数据（.h5ad 格式）→ 后端调用 scBERT 模型进行细胞类型分类 → 前端以交互式散点图展示分类结果 → 专家可对分类结果提出异议并修改。

**差异化定位**：不是传统的 CV 图像标注工具，而是基于空间转录组数据的**人机协同审核平台**。

## 二、技术栈

| 层 | 技术 | 说明 |
|------|------|------|
| 前端框架 | React 19 + TypeScript + Vite | SPA 单页应用 |
| 图表库 | Apache ECharts (Canvas) | 支撑 17,000+ 个点的交互式散点图 |
| 路由 | React Router v6 | 扁平路由结构 |
| 后端框架 | Python 3.13 / FastAPI | RESTful API |
| 数据处理 | h5py | 直接读取 HDF5 格式的 .h5ad 文件 |
| 数据校验 | Pydantic | 严格定义前后端数据结构 |
| 数据库 | 无（MVP 内存缓存） | 后续接入 SQLite |
| scBERT 模型 | Mock（预留接口） | 模型到位后一行替换 |

## 六、后端 API

| 方法 | 端点 | 说明 |
|------|------|------|
| `GET` | `/api/health` | 健康检查 |
| `POST` | `/api/upload` | 上传 .h5ad → mock scBERT → 返回 dataset_id |
| `GET` | `/api/datasets/{id}` | 获取数据集概览（名称、细胞数、类型统计） |
| `GET` | `/api/datasets/{id}/cells` | 获取全部细胞数据（17k+ 条，含坐标、标签、置信度） |

### API 数据格式示例

```json
// POST /api/upload 响应
{
  "dataset_id": "uuid",
  "name": "proseg_after QC_6_11_predict",
  "cell_count": 17045,
  "cell_types": [
    {"name": "Hepatocytes", "count": 12744, "color": "#2a78d6"},
    ...
  ]
}

// GET /api/datasets/{id}/cells 响应
{
  "cells": [
    {
      "cell_id": 42, "x": 10935.2, "y": 6379.9,
      "label": "Hepatocytes", "full_label": "Hepatocytes",
      "prob": 0.3195, "top2_label": "Monocytes / ...",
      "top2_prob": 0.1597, "margin": 0.1598
    },
    ...
  ]
}
```

## 八、设计系统

- **风格**: 简约主义 (Minimalism)，暖白表面 `#fcfcfb`
- **调色板**: 基于 dataviz 方法论验证的 20 色扩展调色板（CVD 色盲友好）
- **字体**: system-ui 原生字体栈，无外部依赖
- **间距**: 8px 基础网格
- **边框**: 极细 `rgba(11,11,11,0.08)` 分隔线
